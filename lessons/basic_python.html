Variables & Syntax

Unlike Java or C++, Python is dynamically typed. You don't declare variable types; the interpreter infers them. Python uses indentation (whitespace) to define code blocks instead of curly braces {}.

F-Strings (String Interpolation)

In professional environments, concatenating strings with + is obsolete. We use f-strings (formatted string literals) for readable, inline variable evaluation.

name = "Dr. Jane Doe"
degree = "Ph.D. in Physics"
grad_year = 2024

# F-string format
greeting = f"{name} graduated with a {degree} in {grad_year}."
print(greeting)

Data Structures

As a graduate, you'll likely use Python for data manipulation. The two foundational structures are Lists (ordered, mutable arrays) and Dictionaries (key-value mappings, equivalent to JSON objects or HashMaps).

Lists & Slicing

Python lists are incredibly flexible. You can extract subsets of lists using slicing syntax: list[start:stop:step].

revenues = [120, 150, 180, 210, 250, 300]

# Slicing: Get Q1 (first three elements)
q1_rev = revenues[0:3] 
print(f"Q1 Revenues: {q1_rev}")

# Get the most recent revenue (negative indexing)
print(f"Latest Revenue: {revenues[-1]}")

Dictionaries

employee = {
    "id": 8472,
    "role": "Data Analyst",
    "skills": ["Python", "SQL", "Tableau"]
}

# Appending to a list inside a dictionary
employee["skills"].append("AWS")
print(f"Updated skills: {employee['skills']}")

Control Flow

Python's logic flows naturally. Note the use of the colon : to open a block, and strict indentation.

List Comprehensions

Instead of writing multi-line for loops to create new lists, Python offers "List Comprehensions" — a highly performant, one-line methodology essential for professional coding.

prices = [10.5, 20.0, 15.75, 50.2]
tax_rate = 0.08

# Standard Loop:
# final_prices = []
# for p in prices:
#     final_prices.append(p * (1 + tax_rate))

# Professional: List Comprehension
final_prices = [round(p * (1 + tax_rate), 2) for p in prices]

print(f"Original: {prices}")
print(f"With Tax: {final_prices}")

Functions & Business Logic

Functions encapsulate logic. In modern Python, we often use Type Hinting (introduced in PEP 484). While Python ignores them at runtime, type hints are crucial for team collaboration and IDE auto-completion.

# Notice the type hints: list, float, and -> float (return type)
def calculate_roi(investment: float, return_value: float) -> float:
    """Calculates the Return on Investment percentage."""
    if investment <= 0:
        return 0.0
        
    roi = ((return_value - investment) / investment) * 100
    return round(roi, 2)

# Using the function
q3_investment = 50000
q3_returns = 62500

profit_margin = calculate_roi(q3_investment, q3_returns)
print(f"The Q3 ROI was {profit_margin}%")
